3. Secure Wipe (Optional)

You can skip this section. The Kali installer's encrypted LVM option will overwrite the drive during installation. This section is only for forensic-grade wiping (e.g., repurposing a laptop that previously held sensitive data).

Boot from the Kali USB. Select "Live (forensic mode)" from the boot menu.

3.1 Identify Your Internal Drive (MANDATORY)

DO NOT SKIP THIS STEP. The dd command in step 3.2 will permanently destroy whatever device you point it at. During a previous deployment, dd wiped the USB installer instead of the internal drive because the device name was assumed instead of verified. You MUST run the identification command below and confirm the correct device BEFORE running any destructive command.
1

MANDATORY: Run this command and read every line of output:

$ sudo lsblk -d -o NAME,SIZE,MODEL,TRAN # # Example output: # NAME SIZE MODEL TRAN # sda 14.9G SanDisk Cruzer usb ← USB STICK (small size, TRAN=usb) # nvme0n1 476.9G Samsung SSD 970 nvme ← INTERNAL DRIVE (large size, TRAN=nvme) # # HOW TO IDENTIFY YOUR INTERNAL DRIVE: # 1. Look at the SIZE column: your internal drive is the LARGE one (hundreds of GB) # 2. Look at the TRAN column: # usb = USB stick (DO NOT WIPE THIS) # nvme = NVMe internal SSD # sata = SATA internal SSD or HDD # ata = SATA internal (older naming) # 3. The USB stick is the SMALL one (8-32 GB) with "usb" in TRAN # # WRITE DOWN your internal drive name: _______________ # (e.g., nvme0n1, sda, sdb — whatever the LARGE non-USB device is) # # IF YOU SEE ONLY ONE DEVICE: your internal drive may not be detected. # Do NOT proceed — check BIOS settings for AHCI/NVMe mode. #

3.2 Execute the Wipe

TRIPLE-CHECK: In every command below, replace /dev/DEVICE with the name you identified above (e.g., /dev/nvme0n1). If your USB shows as sda and your internal drive shows as nvme0n1, then use /dev/nvme0n1. NEVER guess. NEVER assume sda is the internal drive.
2

Overwrite the internal drive with random data:

# Replace /dev/DEVICE with YOUR internal drive name from step 1 # Example: sudo dd if=/dev/urandom of=/dev/nvme0n1 bs=4M status=progress $ sudo dd if=/dev/urandom of=/dev/DEVICE bs=4M status=progress # SANITY CHECKS while it runs: # - This should take 30-120 minutes for a real drive # - You should see progress in GB (e.g., "50 GB copied") # - If it finishes in SECONDS: STOP — you wiped the USB (too small) # - If you see errors immediately: STOP — wrong device # - A 256GB+ drive takes at LEAST 15 minutes
3

For SSDs, also issue a secure erase (uses the drive's built-in erase). Same device name from step 1:

# For NVMe drives: $ sudo nvme format /dev/DEVICE --ses=1 # For SATA drives: $ sudo hdparm --user-master u --security-set-pass dagda /dev/DEVICE $ sudo hdparm --user-master u --security-erase dagda /dev/DEVICE
4

Verify the drive is wiped:

$ sudo hexdump -C /dev/DEVICE | head -20 # Should show random data, NOT filesystem structures or readable text
Note on SSD wear: One pass of random data is sufficient for SSDs. SSDs use wear leveling and TRIM, making multi-pass wiping unnecessary and harmful to drive longevity. The NVMe secure erase command is the definitive solution for SSDs.

4. Install Kali Linux

Choose your installation path. Both paths produce a working system with GNOME and security tools. The difference is when GNOME and tools get installed.

4.1 Boot from USB

1

Insert the bootable USB into the target laptop.

2

Enter BIOS/UEFI setup (usually F2, F12, DEL, or ESC during boot). Configure:

  • Disable Secure Boot (Kali's installer is signed, but kernel modules may not be)
  • Set USB as first boot device
  • Enable UEFI boot (disable Legacy/CSM if present)
3

Save BIOS settings and reboot. The Kali installer menu should appear.

4.2 Run the Installer

4

Select "Graphical install" from the boot menu.

Do NOT select "Expert install". The standard graphical installer is sufficient. Expert mode adds complexity without benefit for this use case.
5

Language, location, keyboard — choose your preferences.

6

Firmware prompt: The installer may warn about missing WiFi firmware (e.g., iwlwifi).
If it asks to load firmware:

  • Select Yes if offered a firmware file to load
  • If it loops back to the same prompt after selecting Yes, select No this time
  • WiFi firmware will be installed after boot (see Post-Install section)
This is normal. Many Intel WiFi chips need firmware that isn't on the installer ISO. The installer works fine without WiFi — it will use Ethernet if connected, or install offline.
7

Network configuration: If the installer says "No default route" or cannot configure the network:

  • Select Yes to continue without a network connection
  • The installer can complete the installation from the USB alone
  • Network will be configured after boot
8

Hostname: dagda (or your preferred name).
Domain: leave blank.

9

Create user account. Use a strong password (16+ characters). This will be your primary login — root is disabled by default on Kali.

REMEMBER THIS PASSWORD. If you forget it, you will need to boot from USB live mode to reset it (see Recovery Instructions). Write it down and store it separately from the laptop.
10

Disk partitioning — select "Guided - use entire disk and set up encrypted LVM".

When asked which disk, select your internal NVMe/SATA drive (the large one, NOT the USB).

Choose encrypted LVM. This encrypts the entire drive with LUKS2 using AES-256-XTS (512-bit key). Without this, anyone who physically accesses the laptop can read the drive. Non-negotiable for a security platform.

Encryption specification:
• Algorithm: AES-256-XTS (two 256-bit AES keys = 512-bit total)
• Key derivation: Argon2id (LUKS2 default, memory-hard — resists GPU brute force)
• This meets FIPS 140-2 Level 1 and NATO RESTRICTED classification requirements
11

Set the LUKS encryption passphrase. Requirements:

  • 20+ characters minimum (longer = stronger against brute force)
  • DIFFERENT from your user password
  • Mix of uppercase, lowercase, numbers, symbols
  • NOT a dictionary phrase, quote, or song lyric
  • Store securely (password manager, sealed envelope in a safe — NOT on the laptop)
  • Losing this passphrase = permanent loss of all data on the drive
12

The installer will overwrite the drive with random data before encrypting. Let it finish — this prevents forensic recovery of old data. May take 30-60 minutes.

13

Partitioning scheme: "All files in one partition" (simplest). Confirm and write changes to disk.

14

Software selection: Check everything that is offered. The standard installer will show options like "standard system utilities" and "Xfce desktop".

Note: The standard Kali installer does NOT offer a GNOME option. It installs Xfce by default. GNOME will be installed after boot (see Post-Install section).
15

GRUB bootloader: The installer will ask about GRUB on EFI systems:

  • "Force GRUB installation to the EFI removable media path?" → Select Yes
  • "Update NVRAM variables to automatically boot into Debian?" → Select Yes
Why both Yes? Some UEFI firmware doesn't respect the standard EFI boot entry. The "removable media path" fallback ensures the laptop can always find GRUB. Updating NVRAM creates the proper boot entry.
16

Installation completes. Remove the USB drive. Reboot.

17

At boot, enter your LUKS passphrase (the 20+ character one, not your user password).

18

Log in with your user credentials (username + the password you set during install).

You are now in Xfce. This is expected for standard Kali. GNOME will be installed in the Post-Install section below.

5. Post-Install Setup

About apt upgrade: After installing GNOME, you MUST run sudo apt -o Dpkg::Options::="--force-overwrite" upgrade -y to bring dependent packages to compatible versions. Without this, GNOME will not start (black screen with blinking cursor).

DO NOT use apt full-upgrade — that is a different, more aggressive command that replaces the kernel and can cause an unbootable BusyBox loop. apt upgrade (without "full-") is safe.

Step 1: Fix GPG Keys + Sources List

Why this is needed: Fresh Kali installs often have missing or outdated GPG keys, and older ISOs may use the wrong component name (non-free instead of the current non-free-firmware). Both issues cause apt update to fail.
# Download the current Kali archive signing key $ wget -q https://archive.kali.org/archive-key.asc -O /tmp/kali-key.asc # Install it to the trusted keyring directory (NOT apt-key — that is deprecated) $ sudo cp /tmp/kali-key.asc /etc/apt/trusted.gpg.d/ # Write the correct sources.list (note: non-free-firmware, not just non-free) $ echo "deb https://kali.download/kali kali-rolling main contrib non-free non-free-firmware" | sudo tee /etc/apt/sources.list # Update package lists $ sudo apt update
If apt update shows GPG errors after this: The key file may not have downloaded correctly. Check your internet connection, re-run the wget command, and try again. Do NOT use apt-key add — it is deprecated and may not work.

Step 2: Install WiFi Firmware (if WiFi is not working)

Check first: If WiFi already works (you can see networks), skip this step. This is only needed if the installer could not load the WiFi firmware. You need Ethernet or a working internet connection to download the package.
# Install Intel WiFi firmware (most common — covers iwlwifi chipsets) $ sudo apt install -y firmware-iwlwifi # Load the kernel module immediately (no reboot needed) $ sudo modprobe iwlwifi # Verify WiFi interface appears $ ip link show # You should see a wlan0 or similar wireless interface # If you have a different WiFi chipset (Realtek, Broadcom, etc.): # sudo apt install -y firmware-realtek # sudo apt install -y firmware-brcm80211

Step 3: Install GNOME Desktop

Why force-overwrite? The kali-desktop-gnome package sometimes has file conflicts with existing wallpaper packages. The --force-overwrite flag resolves this safely by letting the new package's files take priority.
# Install GNOME desktop environment with force-overwrite for wallpaper conflicts $ sudo apt -o Dpkg::Options::="--force-overwrite" install -y kali-desktop-gnome # When asked about display manager, select: gdm3 # (GDM is the GNOME display manager — it provides the graphical login screen)
The display manager prompt is important. If asked to choose between gdm3 and lightdm, select gdm3. It is GNOME's native display manager and required for the GNOME login screen with the session selector gear icon.

Step 3B: Upgrade Dependent Packages (REQUIRED for GNOME to start)

DO NOT SKIP THIS STEP. After installing GNOME, there will be ~2000 packages that need upgrading to compatible versions. Without this upgrade, GNOME will NOT start — you will get a black screen with a blinking cursor after reboot.

Use apt upgrade (safe) — NOT apt full-upgrade (dangerous).
# Upgrade dependent packages so GNOME can start (REQUIRED) # This uses --force-overwrite to handle any file conflicts # This is "apt upgrade" NOT "apt full-upgrade" — they are different commands $ sudo apt -o Dpkg::Options::="--force-overwrite" upgrade -y # This will take 10-30 minutes depending on your internet speed # Do NOT interrupt it. Let it complete fully.
If it fails with error code (1): Run it again:
sudo apt -o Dpkg::Options::="--force-overwrite" --fix-broken install -y
Then re-run the upgrade command above. Package conflicts sometimes need two passes.
# Reboot to apply the new desktop $ sudo reboot

After reboot, at the login screen:

  • Click the gear icon (bottom-right or next to the Sign In button)
  • Select "GNOME" from the session list
  • Enter your password and log in
  • GNOME should now be your desktop — this choice persists across reboots

Step 4: Install Security Tools

# Install essential security tools individually # DO NOT use kali-linux-purple — that package does not exist # DO NOT use kali-linux-everything unless you have 100+ GB free space $ sudo apt install -y nmap wireshark suricata nikto sqlmap gobuster hydra $ sudo apt install -y john hashcat masscan nuclei # Zeek — Kali's apt package has broken libc6 dependency # Build from source (~15 min): $ sudo apt install -y cmake make gcc g++ flex bison libpcap-dev libssl-dev python3-dev swig zlib1g-dev libzmq3-dev libmaxminddb-dev nodejs $ cd /tmp && git clone --recursive --branch v8.1.1 https://github.com/zeek/zeek.git zeek-build $ cd /tmp/zeek-build && ./configure && make -j$(nproc) && sudo make install $ cd ~ && rm -rf /tmp/zeek-build # Volatility — not in Kali apt repos, install via pipx $ sudo apt install -y pipx && pipx install volatility3 # Optional: virtualization (for Arena mode VMs) $ sudo apt install -y libvirt-daemon-system qemu-kvm podman # Optional: audit and hardening $ sudo apt install -y auditd apparmor ufw

Step 5: Enable Firewall

# Set default firewall rules and enable $ sudo ufw default deny incoming $ sudo ufw default allow outgoing $ sudo ufw enable # Verify firewall is active $ sudo ufw status

Step 6: Harden SSH (optional, only if you SSH into this machine)

# Write hardened SSH config (uses tee instead of nano — nano keybindings # can break in some terminal emulators) # Back up original config first $ sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak # Apply hardened settings $ echo "PermitRootLogin no" | sudo tee -a /etc/ssh/sshd_config.d/dagda-hardening.conf $ echo "PasswordAuthentication no" | sudo tee -a /etc/ssh/sshd_config.d/dagda-hardening.conf $ echo "MaxAuthTries 3" | sudo tee -a /etc/ssh/sshd_config.d/dagda-hardening.conf $ echo "X11Forwarding no" | sudo tee -a /etc/ssh/sshd_config.d/dagda-hardening.conf # Restart SSH to apply $ sudo systemctl restart sshd

Step 7: Kernel Hardening

6. Install Dagda

These steps are the same regardless of which installation path you chose above.

1

Install system dependencies:

$ sudo apt install -y python3-pip python3-venv git curl wget jq
2

Clone the repository:

$ cd ~ $ git clone https://github.com/Moonligh7er/Dagda.git $ cd Dagda
3

Create a Python virtual environment and install:

# Create isolated Python environment $ python3 -m venv .venv # Activate it (you'll need to do this each time you open a new terminal) $ source .venv/bin/activate # Install Dagda with all optional dependencies $ pip install -e ".[all]" # Or install groups individually if you prefer: # pip install -e "." # Core only # pip install -e ".[ml]" # ML models (scikit-learn) # pip install -e ".[browser]" # Playwright browser automation # pip install -e ".[reporting]" # PDF report generation
4

Build the Rust security core (optional but recommended):

# Install Rust if not present $ curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh $ source ~/.cargo/env # Build the native module $ cd core $ cargo build --release $ pip install maturin $ maturin develop --release $ cd ..

5

Install Ollama for local AI (optional, enables offline/air-gap operation):

$ curl -fsSL https://ollama.com/install.sh | sh $ ollama pull llama3.2:3b # Small model for HUD $ ollama pull llama3.1:8b # Larger model for orchestrator

Never put API keys in config files. Use environment variables only. Dagda's self-audit checks for plaintext keys in config and will flag them.

Run the setup wizard:

Network configuration # 9. Show a summary

8. Verification Checklist

Run through this checklist to confirm everything is working:

System

  • LUKS encryption prompt appears at boot
  • User login works with correct credentials
  • GNOME desktop is active (not Xfce)
  • WiFi works (if applicable)
  • UFW firewall is active (sudo ufw status)
  • apt update works without GPG errors

Dagda

  • Dagda launches without errors (python -m dagda)
  • SENTINEL mode active (shown in mode indicator)
  • Self-integrity check passes on startup (check audit log)
  • AI responds to chat ("What is the security posture of this system?")
  • Ctrl+Shift+P (PANIC) works — halts AI + fortress + kills comms
  • Ctrl+Shift+K resumes AI after PANIC
  • Ctrl+Shift+I restores comms after PANIC
  • Mode cycling (Ctrl+M) shows: SENTINEL → TRIAGE → INVESTIGATOR → ...
  • Network monitoring detects your own interface
  • Threat feeds pull IoCs (feed_auto_setup + feed_poll_all)
  • ML models loaded (check setup wizard output)
Ongoing maintenance:
• sudo apt update periodically to check for security updates
• Install specific security patches with sudo apt install --only-upgrade PACKAGE
• Do NOT run apt full-upgrade — install updates selectively
• feed_poll_all to refresh threat feeds
• self_integrity_baseline after any Dagda update
• self_audit periodically to check configuration drift

9. Troubleshooting

BusyBox / initramfs Boot Loop

Cause: Usually from apt full-upgrade breaking the kernel or initramfs.
# Option A: Try to fix from BusyBox prompt (initramfs)$ exit # Sometimes this triggers a recovery attempt # Option B: Boot from USB live mode, mount encrypted drive, fix in chroot # (See Recovery Instructions section below) # Once in chroot: (chroot)$ sudo dpkg --configure -a (chroot)$ sudo apt --fix-broken install -y (chroot)$ sudo update-initramfs -u # Option C: If nothing works, reinstall from USB # Your data is lost if you didn't back up

GNOME Black Screen with Blinking Cursor After Install

Cause: GNOME was installed but ~2000 dependent packages were not upgraded to compatible versions. GDM3 reports as "active (running)" but GNOME session fails to start.
# Press Ctrl+Alt+F2 to get a text login prompt # Log in with your username and password # Then upgrade the dependent packages: $ sudo apt -o Dpkg::Options::="--force-overwrite" upgrade -y # If that fails with error code (1): $ sudo apt -o Dpkg::Options::="--force-overwrite" --fix-broken install -y # Then run the upgrade command again # Reboot when complete: $ sudo reboot # If GNOME still won't start, use Xfce temporarily: $ echo "exec startxfce4" > ~/.xinitrc && startx # Then run the upgrade from the Xfce desktop

Password Not Working After Install

# Boot from USB live mode and reset the password # (See Recovery Instructions section below for full chroot procedure) # Once in chroot: (chroot)$ passwd YOUR_USERNAME # Enter new password twice # Exit chroot, unmount, reboot

WiFi Not Showing / No Wireless Interface

# Check if any wireless interface exists $ ip link show # If no wlan interface, install firmware $ sudo apt install -y firmware-iwlwifi # Load the module $ sudo modprobe iwlwifi # Check again $ ip link show # If still nothing, check which chipset you have $ lspci | grep -i network # Then install the matching firmware package

apt update GPG Errors

# Re-download and install the signing key $ wget -q https://archive.kali.org/archive-key.asc -O /tmp/kali-key.asc $ sudo cp /tmp/kali-key.asc /etc/apt/trusted.gpg.d/ # DO NOT use apt-key — it is deprecated # DO NOT use: sudo apt-key adv --keyserver ... # Fix sources.list if needed $ echo "deb https://kali.download/kali kali-rolling main contrib non-free non-free-firmware" | sudo tee /etc/apt/sources.list $ sudo apt update

Package Conflicts (dpkg error code 1)

# Force overwrite conflicting files and fix broken packages $ sudo apt -o Dpkg::Options::="--force-overwrite" --fix-broken install -y

No Internet During Install

This is normal. The installer works offline from the USB. When prompted about network configuration failures or missing default route, select Yes to continue. Configure networking after boot.

nano Not Working / Garbled Input

# Some terminal emulators have keybinding conflicts with nano. # Use the echo + tee pattern instead: # To write a single line to a file: $ echo "your content here" | sudo tee /path/to/file # To append a line to a file: $ echo "your content here" | sudo tee -a /path/to/file # To write multiple lines: $ sudo tee /path/to/file << 'EOF' line 1 line 2 line 3 EOF

GRUB Not Found After Install / Boots to BIOS

# Re-enter BIOS and check: # 1. UEFI boot is enabled (not Legacy/CSM) # 2. The internal drive is in the boot order # 3. Secure Boot is disabled # If GRUB still not found, boot from USB live mode and reinstall GRUB: # (See Recovery Instructions for chroot procedure) (chroot)$ sudo grub-install --target=x86_64-efi --efi-directory=/boot/efi --removable (chroot)$ sudo update-grub

10. Recovery Instructions

If you cannot boot into your installed system (forgot password, BusyBox loop, broken packages, GRUB issues), use this procedure to fix it from a USB live boot.

10.1 Boot into USB Live Mode

1

Insert your Kali bootable USB. Boot the laptop. At the Kali boot menu, select "Live (forensic mode)" or "Live system".

10.2 Identify and Unlock the Encrypted Drive

2

Identify your internal drive (same procedure as the Secure Wipe section):

$ sudo lsblk -d -o NAME,SIZE,MODEL,TRAN # Find your internal drive (large, nvme/sata — NOT usb) # Example: nvme0n1
3

Find the LUKS partition and unlock it:

# List partitions on your internal drive $ sudo lsblk /dev/DEVICE # Look for the largest partition (usually partition 2 or 5) # Example: nvme0n1p5 or sda5 # Open the LUKS encrypted partition # Replace PARTITION with your actual partition (e.g., nvme0n1p5) $ sudo cryptsetup luksOpen /dev/PARTITION crypt_recovery # Enter your LUKS passphrase when prompted # The LVM volume group should now be visible $ sudo vgchange -ay $ sudo lvs # You should see something like: kali--vg root

10.3 Mount and Chroot

4

Mount the filesystem and enter chroot:

# Mount the root filesystem $ sudo mount /dev/mapper/kali--vg-root /mnt # Mount required virtual filesystems $ sudo mount --bind /dev /mnt/dev $ sudo mount --bind /dev/pts /mnt/dev/pts $ sudo mount --bind /proc /mnt/proc $ sudo mount --bind /sys /mnt/sys # Mount the EFI partition (usually partition 1) $ sudo mount /dev/DEVICEp1 /mnt/boot/efi # Enter the chroot $ sudo chroot /mnt /bin/bash

10.4 Fix the Problem

5

Now you are inside the installed system. Apply the fix you need:

Reset a forgotten password:

(chroot)$ passwd YOUR_USERNAME # Enter new password twice

Fix broken packages (BusyBox loop):

(chroot)$ dpkg --configure -a (chroot)$ apt --fix-broken install -y (chroot)$ update-initramfs -u

Reinstall GRUB:

(chroot)$ grub-install --target=x86_64-efi --efi-directory=/boot/efi --removable (chroot)$ update-grub

Fix GPG keys (if apt is broken):

(chroot)$ wget -q https://archive.kali.org/archive-key.asc -O /tmp/kali-key.asc (chroot)$ cp /tmp/kali-key.asc /etc/apt/trusted.gpg.d/ (chroot)$ echo "deb https://kali.download/kali kali-rolling main contrib non-free non-free-firmware" > /etc/apt/sources.list (chroot)$ apt update

10.5 Exit and Reboot

6

Clean up and reboot:

# Exit the chroot (chroot)$ exit # Unmount everything $ sudo umount /mnt/boot/efi $ sudo umount /mnt/sys $ sudo umount /mnt/proc $ sudo umount /mnt/dev/pts $ sudo umount /mnt/dev $ sudo umount /mnt # Close the LUKS volume $ sudo vgchange -an $ sudo cryptsetup luksClose crypt_recovery # Remove USB and reboot $ sudo reboot

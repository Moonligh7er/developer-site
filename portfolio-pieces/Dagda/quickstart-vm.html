VM Quick Start Guide — Try Dagda in 15 Minutes
No drive wiping. No dedicated hardware. Just a VM.
This guide is for evaluation and development. For production deployment on dedicated hardware, see the full Laptop Deployment Guide.
~15 minutes from start to running

What You Need

  • A computer with 8+ GB RAM (16 GB recommended)
  • VirtualBox, VMware, or Hyper-V installed
  • 20 GB free disk space
  • Internet connection

Step 1: Download Kali Linux

1

Download the pre-built Kali VM image (easiest — no installation needed):

# Go to: https://www.kali.org/get-kali/#kali-virtual-machines # Download the image for your hypervisor: # - VirtualBox: .ova file (~3 GB) # - VMware: .7z file (~3 GB) # - Hyper-V: .vhdx file (~3 GB) # Default credentials: kali / kali
Why not Kali Purple specifically? The pre-built VM images are standard Kali. You can install the Purple tools after boot with sudo apt install kali-linux-purple. Or skip it entirely — Dagda works on any Kali (or Debian/Ubuntu).

Step 2: Configure the VM

2

Import the VM image into your hypervisor, then adjust settings:

SettingMinimumRecommended
RAM4 GB8 GB
CPUs24
Disk20 GB40 GB
NetworkNATBridged (for network scanning)
3

Boot the VM and log in with kali / kali. Open a terminal.

Step 3: Install Dagda

4

Update the system and install prerequisites:

$ sudo apt update && sudo apt upgrade -y $ sudo apt install -y python3-pip python3-venv git
5

Clone and install Dagda:

$ cd ~ $ git clone https://github.com/Moonligh7er/Dagda.git $ cd Dagda # Create virtual environment $ python3 -m venv .venv $ source .venv/bin/activate # Install Dagda with all optional dependencies $ pip install -e ".[all]"
6

(Optional) Install common security tools that Dagda orchestrates:

# These are already on Kali, but verify: $ sudo apt install -y nmap nuclei nikto gobuster sqlmap \ masscan hydra john hashcat smbclient # Optional: Kali Purple defensive tools $ sudo apt install -y suricata zeek
7

(Optional) Install local AI for offline/air-gap operation:

$ curl -fsSL https://ollama.com/install.sh | sh $ ollama pull llama3.2:3b # Small model for HUD (~2 GB) # Or set Claude API key for cloud AI: $ export ANTHROPIC_API_KEY="sk-ant-..."

Step 4: Run the Setup Wizard

8

Run the interactive setup wizard:

$ python -m dagda --setup # The wizard walks you through: # 1. Security assessment of the VM # 2. Profile selection (Standard recommended for VMs) # 3. Tool detection (finds nmap, nuclei, etc.) # 4. Optional tool installation # 5. OS optimization # 6. ML model check # 7. Summary

Step 5: Launch

9

Start Dagda:

$ python -m dagda # You should see the TUI with: # - Mode indicator: SENTINEL # - Safeguard level: 3 (BALANCED) # - Chat panel on the left # - Tool output on the right # - Feed panel at the bottom
You're running. SENTINEL mode is active. Ask the AI anything:

"What is the security posture of this system?"
"Scan localhost for open ports"
"Show me the threat feeds"

VM Limitations

These features require bare metal (see the full Deployment Guide):

FeatureVM StatusWhy
eBPF monitoringUnavailableNeeds host kernel access
Wireless attacksUnavailableNeeds physical WiFi card
RF KillNo effectVirtual NIC, no radios
Arena VMsSlow / limitedNested virtualization
ISOLATE (comms kill)VM networking onlyDoesn't affect host

Everything else — all 640 tools, all 9 modes, AI orchestration, threat feeds, red team suite, Morrigan fleet, self-security — works identically in a VM.

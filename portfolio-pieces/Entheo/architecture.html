Architecture & Infrastructure Map

How This App Is Built

Entheo is split into three pieces that work together: a website, a server (the "brain"), and a mobile app. Here's what each part does and what outside services they talk to.

Website

What users see in their browser. Built with Next.js and hosted on Vercel. Can be installed as a phone app (PWA).

Vercel Dashboard →

Server

The backend brain. Handles logins, saves data, talks to AI, sends emails. Runs in a Docker container on Railway.

Railway Dashboard →

Mobile App

iOS & Android app built with Expo (React Native). Same features as the website, plus push notifications and native feel.

Expo Dashboard →

Outside Services (What You Pay For)

PostgreSQL Database Required

This is where ALL user data lives — accounts, journal entries, mood analyses, sessions, everything. It's a traditional database provided automatically by Railway when you add their PostgreSQL plugin. Railway handles backups, scaling, and uptime.

Managed by: Railway · Free tier available, paid for production

Claude AI (Anthropic) Optional

The AI that reads journal entries and figures out your mood, analyzes dreams through a Jungian lens, generates personalized writing prompts, spots patterns over time, builds a psychological profile, and powers the integration companion chat. Without it, the app still works — you just don't get AI features.

Three different Claude models are used depending on the task: a fast cheap one (Haiku) for simple scoring, a balanced one (Sonnet) for creative writing and chat, and the most powerful one (Opus) for deep pattern analysis (Premium users only).

Managed at: Anthropic Console · Pay per use (~$0.25-3.00/1M tokens depending on model)

OpenWeatherMap Optional

Gets the current weather and 12-hour forecast based on the user's location. This is attached to journal entries so you can correlate your mood with weather patterns. Also powers weather-locked daily quests. Without it, weather features are simply hidden.

Managed at: OpenWeatherMap Dashboard · Free tier: 1,000 calls/day

Stripe (Web Payments) Optional

Handles subscription payments on the website. Users click "Subscribe," get redirected to a Stripe checkout page, pay, and come back with an active subscription. Stripe also provides a portal where users can manage their plan, update payment info, or cancel.

Managed at: Stripe Dashboard · 2.9% + $0.30 per transaction

RevenueCat (Mobile Payments) Optional

Handles in-app purchases on iOS and Android. It sits between the app and Apple/Google's payment systems, making subscriptions work the same way on both platforms. When a user subscribes on mobile, RevenueCat tells our server to upgrade their account.

Managed at: RevenueCat Dashboard · Free up to $2,500/mo revenue, then 1%

Email (SMTP) Optional

Sends transactional emails: account verification, password resets, and weekly digest summaries. Without an email provider configured, these messages just print to the server console (useful for development). Recommended provider: Resend.

Managed at: Resend Dashboard · Free: 100 emails/day

Sentry (Error Tracking) Optional

Catches crashes and errors in real-time across both the website and server. When something breaks, Sentry captures the error with full context (what the user was doing, the stack trace, etc.) and can notify you via email or Slack.

Managed at: Sentry Dashboard · Free: 5K errors/mo

S3 / R2 (File Storage) Optional

Cloud storage for large files — specifically ceremony audio recordings. Without it, audio is stored as text inside the database (works but doesn't scale). When configured, audio files are uploaded to a cloud bucket instead, and only a reference link is stored in the database.

Managed at: AWS S3 or Cloudflare R2 · R2: Free up to 10GB

Plausible Analytics Optional

Privacy-friendly website analytics. Unlike Google Analytics, it doesn't use cookies and doesn't track individual users. Shows you page views, referrers, and general traffic patterns without any privacy concerns.

Managed at: Plausible Dashboard · Starts at $9/mo

How Data Is Protected

Passwords are scrambled using bcrypt (one-way hash — even we can't read them).

Journal entries, dreams, and drafts are encrypted before being stored in the database using AES-256-GCM (military-grade encryption). Each entry has its own random key derived from a master secret. Even if someone stole the database, they couldn't read the content without the master key.

Login sessions use two tokens: a short-lived one (15 minutes) for normal requests and a longer one (7 days) for staying logged in. Both are stored as httpOnly cookies that JavaScript can't read — protecting against XSS attacks.

Two-factor authentication is available via authenticator apps (Google Authenticator, Authy, etc.).

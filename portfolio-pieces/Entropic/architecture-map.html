Entropic Tarot — Architecture Map

Complete system architecture, external services, accounts, and maintenance costs

How the App Is Built

Entropic Tarot is a web application that also runs as a mobile app on iPhone and Android. It's built with one codebase that works everywhere — your web browser, the App Store, and Google Play.

The app runs almost entirely on the user's device. The only time it talks to an external server is when the user requests an AI interpretation of their reading. Everything else — card shuffling, entropy collection, card images, esoteric data, saved readings — happens locally.

AI Interpretation

Active Required for AI feature

Anthropic Claude API — The AI Reader

When a user presses "Interpret My Reading," the app sends their card data to Anthropic's Claude AI, which writes a personalized interpretation. The AI sees the cards, their positions, Qabalah correspondences, and current planetary positions — but never any personal information about the user.

What it does
Generates narrative tarot interpretations from card spreads
Model used
Claude Sonnet 4.6 (fast, high quality, cost-effective)

API key format

Data Storage

Browser localStorage — User's Saved Readings

All saved readings, audio preferences, and onboarding state are stored in the user's browser on their own device. Nothing is sent to a server. If the user clears their browser data, their readings are gone.

What's stored
Saved readings (cards, positions, AI interpretations), audio mute preference, onboarding completion flag
Where
User's browser, on their device only
Cost
Free (built into every browser)
Limitations
~5-10MB per domain; no sync between devices

API Endpoints

POST /api/interpret Active

Streams an AI-generated tarot interpretation via Server-Sent Events.

Auth
None (rate-limited by IP + User-Agent fingerprint)
Rate Limit
5 requests / 24 hours / IP
Request Body
Zod-validated: { spreadType, deckName?, cards[], planetaryPositions?, timestamp }
Max Body
10KB (express.json limit)
Max Prompt
50,000 characters
Response
SSE stream: data: {"text":"..."}\n\n ... data: [DONE]\n\n
Timeout
60 seconds
Model
process.env.ANTHROPIC_MODEL || 'claude-sonnet-4-6-20250514'
Max Tokens
2,000
Error Format
data: {"error":"safe message"}\n\n (never exposes raw API errors)

GET /api/interpret/status Active

Returns remaining daily interpretation count for the requesting IP.

Response
{ remaining: number, dailyLimit: 5, resetsIn: ms }

Cryptographic Pipeline (Client-Side)

Microphone (200ms) → SHA-256 → Entropy Pool
Camera (300ms) → SHA-256 → Entropy Pool
Touch (on interaction) → SHA-256 → Entropy Pool
Entropy Pool → + crypto.getRandomValues(32) → HMAC-DRBG (NIST SP 800-90A) → Fisher-Yates + Rejection Sampling → 78-card shuffled deck

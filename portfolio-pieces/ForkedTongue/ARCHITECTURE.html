System Overview

What Is Forked Tongue?

Forked Tongue is a language learning app with three main parts: a website, a mobile app, and a server that powers both. Users talk to AI characters who help them learn languages through conversation.

Think of it like having 16 different conversation partners, each with their own personality, teaching style, and expertise. The AI adapts to how you learn, remembers your weaknesses, and adjusts difficulty in real time.

The Three Parts

Website — What users see in their browser. Built for speed and works on any device.

Mobile App — A native app for iPhone and Android, distributed through the App Store and Play Store.

Server (API) — The backend brain. Handles user accounts, talks to the AI, tracks progress, and manages subscriptions.

The Hidden Packages

Shared — Code that both the website and mobile app need (types, constants, utilities).

AI — Wrappers around Claude (conversation), ElevenLabs (speech), and Deepgram (voice input).

Characters — The 16 AI conversation partners: their backstories, teaching styles, and conversation rules.

LNN — The Linguistic Neural Network — a custom engine that maps what a user knows, finds gaps, and drives adaptive difficulty.

External Services

Services at a Glance

Forked Tongue relies on 10 external services. Here is what each one does in plain English:

Anthropic (Claude) — The AI brain. Every conversation, correction, and analysis flows through Claude. It is the single most important service.

ElevenLabs — Turns text into lifelike speech for premium users. Free users get Kokoro (an in-browser voice engine that costs nothing).

Deepgram — Turns spoken words into text for premium users. Free users get the browser's built-in speech recognition.

PostgreSQL — The database. Stores user accounts, conversations, messages, points, and progress.

Redis — A fast cache. Handles rate limiting, session management, and temporary data.

Stripe — Processes credit card payments for website subscriptions.

RevenueCat — Handles in-app purchases on iPhone and Android so we don't deal with Apple/Google billing directly.

Google AdSense — Shows ads to free web users to generate revenue.

Google AdMob — Shows ads to free mobile users.

Sentry — Catches errors in production and alerts us. Not yet configured.

Data Flow Diagram

How a Conversation Works

1. You type or speak a message in the app.

2. The message is sent to our server.

3. The server checks your login, applies rate limits, then builds a massive prompt for the AI. That prompt includes your character's personality, your learning history, your weak points, and the entire conversation so far.

4. The AI (Claude) generates a response, which streams back word-by-word in real time.

5. The server saves the message, updates your progress, and (for premium users) also sends the text to ElevenLabs to create speech audio.

6. If you used voice input, Deepgram converted your speech to text before step 2.

7. Everything is cached in Redis for speed and stored permanently in PostgreSQL.

User (Web / Mobile)
↓ HTTPS / WSS
API Server (Express :3001)
↓ Auth (JWT + Redis blacklist)
↓ Rate Limiter (Redis + in-memory fallback)
↓ Route Handler
→ Prisma → PostgreSQL :5433
→ Claude API (conversation / analysis)
→ Redis :6379 (caching / sessions)
↓ WebSocket (real-time streaming)
→ Claude streaming → token-by-token delivery
→ Deepgram STT (voice input)
→ ElevenLabs TTS (voice output)

System Prompt Composition

How the AI "Knows" You

Every time you send a message, the app assembles a detailed instruction set for the AI from 15 different sources. This is what makes each conversation feel personalized and adaptive rather than generic.

It includes the character's personality, your learning mode, your weak spots, past conversation summaries, what teaching strategies work best for you, and even your pronunciation trouble areas. All of this is stitched together dynamically before every API call.

15-Layer Prompt Assembly

Each conversation request assembles the system prompt from these sources, in order:

1
Character Template
Persona, traits, interests, speaking style, native language
2
Conversation Mode Instructions
Chill / Nudge / Immersion / Tutor — controls correction density and target language ratio
3
LNN Adaptive Fragment
Weak nodes, error-prone concepts, suggested practice targets from the Linguistic Neural Network
4
Conversation Memory
Summarized history of the current conversation thread
5
Character Memory
Cross-session personality memory — things the character "remembers" about the user
6
Language Triangulation
Multi-language comparisons (e.g., "In Spanish you already know this pattern; French uses it differently")
7
Teaching Notes
What strategies work best for this specific user (cached in Redis)
8
Linguistic Insights
Past correction discoveries — recurring error patterns
9
Cross-User Difficulty Patterns
Universal hard concepts aggregated from all users (Redis cache)
10
Pronunciation Profile
Phoneme weaknesses specific to this user's native language → target language pair
11
Emotional State Guidance
Frustration / confidence detection — adjusts encouragement level and difficulty
12
Immersion Moment Trigger
10% random chance, memory-personalized — brief full-immersion bursts
13
Contextual Scenario Suggestion
5% chance, topic-matched — "Let's pretend you're ordering at a cafe"
14
Active Recall Challenge
20% chance, spaced repetition quiz on previously learned items
15
Content Filter
Post-response safety net — checks output for policy compliance

Monthly Cost Projections

What Does It Cost to Run?

For 1,000 users (900 free, 80 Plus, 20 Premium), the app costs roughly $2,200–$2,500 per month to operate. The biggest expense by far is the Claude AI API. Voice services (ElevenLabs, Deepgram) only apply to Premium users.

Revenue from subscriptions and ads should bring in roughly $2,640 per month, making the app profitable at this scale.

The key insight: free users are cheap (Kokoro TTS is free, Web Speech API is free, and cached Haiku prompts are very affordable). Premium users cost more per-head but pay enough to cover it.

Assumptions

1,000 users: 900 Free, 80 Plus ($14.99/mo), 20 Premium ($49.99/mo)

Messages/day: Free = 15, Plus = 60, Premium = 150

Claude cached Haiku cost per message: ~$0.0025 (with prompt caching)

Claude Sonnet invocations: ~10% of messages at ~$0.0075 each

Deployment Checklist

Everything Needed to Go Live

This is the full checklist. Work through it top to bottom.

Pre-Launch Requirements

Each item must be completed before production deployment.

Infrastructure

  • Set up production PostgreSQL (Supabase / Neon / RDS)
  • Set up production Redis (Upstash / ElastiCache / Redis Cloud)
  • Configure HTTPS / SSL certificates
  • Deploy API server to cloud (Docker / Railway / Render)
  • Deploy web frontend to CDN (Vercel / Netlify / Cloudflare Pages)
  • Set up monitoring and alerting

API Keys & Accounts

  • Get Claude API key (console.anthropic.com)
  • Get ElevenLabs API key for Premium tier (elevenlabs.io)
  • Get Deepgram API key for Premium tier (console.deepgram.com)
  • Set up Stripe account + create products/prices (stripe.com)
  • Set up RevenueCat + connect App Store / Play Store (revenuecat.com)
  • Set up Google AdSense account (adsense.google.com)
  • Set up Google AdMob account (admob.google.com)
  • Set up Sentry project (sentry.io)

Mobile

  • Build mobile app with EAS Build
  • Submit to Apple App Store (requires Apple Developer account, $99/year)
  • Submit to Google Play Store (requires Google Developer account, $25 one-time)
  • Configure in-app purchase products in App Store Connect and Google Play Console
  • Set up ASO metadata (screenshots, descriptions, keywords)

Pre-Flight Checks

  • Run database migrations on production (prisma migrate deploy)
  • Verify all environment variables are set
  • Test Stripe webhooks with stripe listen --forward-to
  • Test RevenueCat webhook delivery
  • Load test API with expected traffic volume
  • Verify CORS configuration for production domain
  • Confirm cookie consent and TCF v2.3 compliance
  • Review privacy policy and terms of service

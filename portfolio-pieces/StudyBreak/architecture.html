What Each Service Does

Services Explained

☍
Supabase
Database, Login, File Storage Required

Think of Supabase as the "filing cabinet" for everything. It stores all user accounts, classes, flashcards, assignments, chat history, and uploaded documents. It also handles login/signup (creating accounts, verifying passwords, issuing login tokens). When you upload a PDF, it goes into Supabase Storage.

Dashboard Docs
✨
Anthropic Claude API
AI Features Required

Claude is the "brain" behind every AI feature: the tutor chat, flashcard generation, document summaries, exam prep quizzes, college advising, assignment solving, and more. We use two models: Sonnet 4.6 (fast, for most tasks) and Opus 4.6 (smarter, for premium users on Scholar+ plans). You pay per message based on how long the input and output are.

Console Docs
💳
Stripe
Web/Desktop Payments Required

Stripe handles all subscription billing on the website: collecting credit cards, charging monthly/yearly, managing upgrades/downgrades, and sending receipts. You create "Products" and "Prices" in the Stripe dashboard, then the app uses their IDs to know what to charge. Stripe takes 2.9% + 30¢ per transaction.

Dashboard Docs
🎤
Deepgram
Lecture Transcription Required

When a student records a lecture, the audio is sent to Deepgram which converts speech to text. It can also identify different speakers ("diarization"). Paid users get a monthly minute allowance (Student: 60 min, Scholar: 300 min, Genius: 600 min).

Console Docs
🔒
RevenueCat
App Store Billing Required

When someone subscribes through the iOS App Store or Google Play Store, Apple/Google handle the payment, and RevenueCat syncs the subscription status back to our server. It's a middleman that makes in-app purchases work without building custom receipt validation.

Dashboard Docs
🔔
Firebase Cloud Messaging
Push Notifications Optional

Firebase sends push notifications to users' phones and browsers — things like "You have 15 overdue flashcards" or "Your Calc assignment is due tomorrow." If Firebase isn't configured, the app still works, users just won't get push alerts.

Console Docs
🔈
Text-to-Speech (3 providers)
Read Aloud Feature Optional

When a student clicks "Read Aloud" on a document or summary, the text is converted to spoken audio. We have a fallback chain: Genius users get ElevenLabs (best quality), Scholar gets Google Neural voices, Student gets Kokoro (open-source via DeepInfra), and Free users get their browser's built-in voice (no server needed).

ElevenLabs Google TTS DeepInfra
🐛
Sentry
Error Tracking Optional

Sentry automatically catches errors (crashes, unhandled exceptions) on both the server and in users' browsers, then sends you an alert with the full stack trace and context. Extremely useful for finding bugs in production. Without it, you'd have to dig through server logs manually.

Dashboard
📊
PostHog
Analytics & Feature Flags Optional

PostHog tracks how users interact with the app — which pages they visit, which features they use, where they drop off. It also supports "feature flags" (turning features on/off for specific users without deploying code). Useful for understanding usage patterns and A/B testing.

Dashboard
🌎
Render.com
Web Hosting Required

Render hosts the entire application — both the server that handles API requests and the built website files. You push code to GitHub and Render automatically builds and deploys it. The Starter plan ($7/mo) keeps the server always running (no cold starts).

Dashboard
🎬
OpenStreetMap + CARTO
Campus Map Tiles Optional

The Campus Map feature shows an interactive map of the student's university. The map background images ("tiles") come from CARTO's free servers, and building data is pulled from OpenStreetMap's Overpass API. Both are free and open-source.

OpenStreetMap CARTO
🌦
OpenWeatherMap
Weather Data Optional

Provides current weather for the student's campus, used to generate context-aware study suggestions (e.g., "It's rainy today — great day for a library session"). Falls back gracefully if not configured.

Where Data Lives

Data Storage

🗃
PostgreSQL Database
66 tables · Hosted on Supabase

Every piece of structured data — users, classes, flashcards, assignments, chat messages, quiz results, study plans, achievements, and more. Connected via Drizzle ORM (a type-safe query builder).

📁
Supabase Storage
2 buckets: documents, study-sessions

Uploaded files (PDFs, images, audio) stored as objects. Images auto-optimized (resized to 2048px, converted to WebP). Legacy files served from local /uploads/ directory as fallback.

📱
IndexedDB (Browser)
Offline support via Dexie.js

Flashcards, classes, and assignments cached locally in the browser for offline access. A sync queue tracks pending changes and replays them when the user reconnects.

⚡
In-Memory Cache
AI responses · 10K entries max · 1hr TTL

Frequently-requested AI results (embeddings, summaries, tool outputs) cached in server memory to avoid re-calling Claude for identical requests. Automatically cleaned every 5 minutes.

The largest variable cost is Anthropic Claude. At 500 users averaging 10 AI requests/day, expect ~$80–250/mo depending on Opus usage. At 5,000 users, expect ~$800–2,500/mo. The AI cache reduces redundant calls by ~30%.

Optional services (Sentry, ElevenLabs, Google TTS) can be dropped to reduce costs by ~$50/mo. The app degrades gracefully without them.

Revenue vs Cost at Scale

At 500 users with 10% paid conversion (50 paid users, avg $10/mo) = $500/mo revenue vs ~$300/mo cost = profitable at ~50 subscribers.
At 5,000 users with 10% conversion (500 paid) = $5,000/mo revenue vs ~$1,500/mo cost = strong margin at scale.
